# Quiver target/compound panel index

Single catalog of every target/compound panel data file in this repo, so future
model tests pick a known-good panel instead of re-deriving one. Generated by the
Models Claude (`models` branch), 2026-06-14.

## How to use this

- **One row = one data file.** Paths are repo-relative (root =
  `/Users/rohanaryagondi/mammal-models-wt`). Files are referenced, **not copied**
  — read them in place.
- **Ready-to-use binder/decoy panels** carry `label` (1 = binder, 0 = decoy),
  `smiles`, and `protein_seq` per row → drop straight into a DTI / selectivity
  eval. These are the JSONs under `docs/boltz_handoff/data/` and
  `aws/crossmodal_panels.json`.
- **Result/eval JSONs** (`results/*_quiver_targets_*.json`,
  `results/compare_conplex_nav_offtarget_*.json`, `results/offtarget_ube3a_*.json`)
  embed the same compounds *plus* scores — useful as a labeled source and as a
  baseline to beat, but the canonical clean panels are the `docs/boltz_handoff/`
  set.
- **Protein-only panels** (`results/big_panel.json`,
  `results/phase5_crispr_panel_*.json`) have sequences/families but no compounds —
  Track 1 (family clustering) and CRISPR-N family-coherence work, not DTI.
- **`results/dti_train_data_per_target.csv`** is per-target *metadata* (pair
  counts per UniProt), not compound-level data — use it to pick which targets
  have enough BindingDB training signal, not as a binder panel.
- Sequences in the boltz-handoff panels are full-length; the MAMMAL DTI head
  truncates targets to 1250 aa internally (see `CLAUDE.md`).
- `label` convention is consistent across all panels: **1 = binder/active,
  0 = decoy/inactive**.
