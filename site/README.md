# Sapphire site

Two pages:

- **`index.html` — the Console (main).** The front face of the orchestrator: ask a question and watch
  the firm run — PLAN → DISCOVER (fact dossier) → VALIDATE → CONSULT (two-round roundtable) → SYNTHESIZE.
- **`explainer.html` — the full flow.** The foundation behind it: methodology, the gate→boost system
  flow (animated on the real Nav1.8 SCN11A #7→#1 run), the 3-layer data map, the 16-capability dashboard.

## Run it

**Live (Claude under the hood — your subscription):**
```bash
python ../sapphire-orchestrator/serve.py     # http://localhost:8077  (serves the site + the live brain)
```
The two shipped scenarios run instantly from the engine ($0). A **novel** question routes to **Claude
Code headless on your subscription** (no API key) and comes back as a full run. The status badge shows
`live · Claude (subscription)` when the bridge is up.

**Static (anywhere, no bridge):**
```bash
python -m http.server 8077      # or open index.html directly
```
Without the bridge the Console plays the canned scenarios and shows the engagement plan for any other
query (badge: `demo`). Everything still works — you just don't get the live brain.

## Files

| File | Role |
|---|---|
| `index.html` | The Console (main page). Loads `orchestrator_data.js` + `console.js`. |
| `explainer.html` | The full-flow explainer. Loads `data.js` + `app.js`. |
| `console.js` | Drives the Console: planner mirror, health-check the bridge, live/canned rendering. |
| `app.js` | Drives the explainer: methodology, cascade animation, filters, capability drawer. |
| `styles.css` | The dark "electrophysiology instrument" theme. |
| `data.js` | Auto-generated from `../docs/foundation/capability_map.xlsx` by `build_site_data.py`. |
| `orchestrator_data.js` | Auto-generated by `build_orch_data.py`, which **runs the engine** — so the Console renders exactly what `run.py` produces. |

## Regenerate the data
```bash
python ../_build/build_site_data.py     # capability dashboard ← docs/foundation/capability_map.xlsx (needs openpyxl)
python ../_build/build_orch_data.py     # Console scenarios    ← runs sapphire-orchestrator/orchestrator.py
```
