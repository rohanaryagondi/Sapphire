# Real Internal Moat — Implementation Plan (Loka CNS_DFP → Sapphire)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** Replace Sapphire's MOCK internal moat with a real data layer over Loka's
`CNS_DFP_distance_20251215.parquet` (38.4M perturbation↔perturbation EP-signature distances), exposing
similar (mimic) / opposite (rescue) perturbation lookups, wired into the engine with provenance `moat-real`.

**Architecture (stdlib-only runtime):** A one-time **ingest** (`_build/build_moat_db.py`, may use pyarrow —
a build tool, not a runtime dep) streams the parquet and materializes **top-K neighbors per query
perturbation** into a **SQLite** DB (`sqlite3` is stdlib) — mirroring Loka's RDS materialized-view approach.
The **runtime moat client** (`sapphire-orchestrator/moat/`) is pure stdlib (`sqlite3`), so the engine stays
dependency-free and tests run offline against a tiny temp SQLite fixture (no pyarrow, no parquet needed).

**Science semantics (verified by profiling 2026-06-21):** direction column is `Original`; query types are
genes, ref types are `gene` + `compound`; cosine ∈ [0,2]. **similar = smallest cosine** (correlated EP →
mimic), **opposite = largest cosine** (anti-correlated EP → rescue). Always exclude self (`ref == query`).
Sanity check: similar(TSC2) → TSC1 (cos 0.084), the TSC/mTOR complex.

## Data contract (the SQLite the ingest writes; the client reads)
```sql
CREATE TABLE neighbors(
  query TEXT, query_type TEXT,
  ref TEXT, ref_type TEXT, ref_dose TEXT,
  effect TEXT,            -- 'similar' | 'opposite'
  rank INTEGER,           -- 1..K within (query, effect)
  cosine REAL, euclidean REAL
);
CREATE INDEX ix_neighbors_q ON neighbors(query, effect, ref_type, rank);
CREATE TABLE moat_meta(key TEXT PRIMARY KEY, value TEXT);   -- source, built_at, k, n_queries, n_rows
```

### Task 3: Ingest build script (`_build/build_moat_db.py`)  — pyarrow allowed here
**Files:** Create `_build/build_moat_db.py`; Test: `sapphire-orchestrator/moat/tests/test_ingest.py` (tests the pure top-K logic only, with an in-memory list — no pyarrow/parquet in the test).
**Interfaces (produce):**
- A pure, testable function `topk_neighbors(rows, k) -> list[dict]` where `rows` is an iterable of dicts
  `{query, query_type, ref, ref_type, ref_dose, cosine, euclidean}`. It groups by `query`, **excludes
  `ref == query`**, and returns neighbor records for both effects: the `k` smallest-cosine (`effect="similar"`,
  rank 1..k ascending) and the `k` largest-cosine (`effect="opposite"`, rank 1..k descending), each record
  `{query(UPPER), query_type, ref, ref_type, ref_dose, effect, rank, cosine, euclidean}`. Deterministic
  tie-break by `ref` name.
- A `main()` that: reads the parquet via `pyarrow.dataset` **streaming in batches** (low memory), feeds rows
  through the same top-K logic (maintain per-query bounded heaps, K from env `MOAT_K` default 50), writes the
  `neighbors` + `moat_meta` tables + index to the SQLite at env `SAPPHIRE_MOAT_DB` (default `<repo>/RohanOnly/moat/moat.sqlite`),
  reading the parquet from env `MOAT_PARQUET` (default the Loka path `Career/Quiver/Sapphire/Loka - Shared Folder/Data/CNS_DFP_distance_20251215.parquet`).
  Prints progress + a sanity block (similar/opposite for `TSC2`). Uses `heapq` for bounded per-query heaps so 38M rows stream in bounded memory.
